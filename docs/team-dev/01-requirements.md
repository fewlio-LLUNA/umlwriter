# 要件定義書 — UML パッケージ（フォルダ型の枠）

> `/req-define` の成果物。ユーザー承認後に確定。設計の入力になる。

最終更新: 2026-07-10
ステータス: **承認済み**（2026-07-10）

## 1. 目的・背景
- 複数のクラスを「パッケージ」として視覚的にグループ化し、図の構造を分かりやすくしたい。
- UML のパッケージ記法（左上に小さなタブが付いたフォルダ型の矩形）で囲めるようにする。

## 2. ユーザーと利用シーン
- 想定ユーザー: クラス図を描く本人（Miliba）。
- 主な利用シーン: 関連の強いクラス群を枠で囲み、サブシステム／モジュールの区切りを示す。

## 3. スコープ
### やること（In）
- ツールバーの「パッケージ追加」ボタンでパッケージ枠を1つ追加する。
- パッケージはフォルダ型（本体の矩形＋左上のタブ）で描画する。
- パッケージは **見た目の枠だけ**。中のクラスとは独立で、枠を動かしてもクラスは動かない。
- パッケージを **手動リサイズ**できる（選択時に角・辺のハンドルをドラッグ）。
- パッケージをドラッグで移動できる。
- パッケージは **クラスより背面** に描画し、クラスのクリック・選択・編集を邪魔しない。
- パッケージ名ラベルを持ち、タブ部分に表示。Inspector で編集できる。
- パッケージを選択・削除できる。
- JSON / localStorage に永続化し、書き出し／読み込みで round-trip する。
- 画像書き出し（SVG / PNG）にパッケージ枠を含める。

### やらないこと（Out）
- 真のグループ化（親子関係・枠を動かすと中身も追従・枠内へのクラス拘束）。※今回は不採用
- 中のクラスへの自動フィット（サイズは手動のみ）。
- パッケージの入れ子（ネスト）。
- パッケージ単位の折りたたみ／一括移動などの高度操作。
- クラスを選択して「パッケージ化」する導線（今回はボタン追加のみ）。

## 4. 機能要件
| ID | 機能 | 概要 | 優先度 |
|----|------|------|--------|
| F1 | パッケージ追加 | ツールバーのボタンで既定サイズのパッケージを配置 | MUST |
| F2 | フォルダ型描画 | 本体矩形＋左上タブ。本体は（ほぼ）透明で背面のクラスが見える | MUST |
| F3 | 移動 | ドラッグで位置変更（自動保存に乗る） | MUST |
| F4 | 手動リサイズ | 選択時に角・辺ハンドルで幅・高さを変更 | MUST |
| F5 | 背面描画 | クラスより低い z で描画し、クラス操作を妨げない | MUST |
| F6 | 名前編集 | タブに名前表示、Inspector で編集 | MUST |
| F7 | 選択・削除 | 選択で Inspector に出す。Delete/Backspace で削除 | MUST |
| F8 | 永続化 | Diagram に packages を追加し JSON/localStorage を round-trip | MUST |
| F9 | 画像書き出し | SVG / PNG にパッケージ枠を含める | SHOULD |

## 5. 非機能要件
- ブラウザ完結・バックエンドなしの既存構成を維持。
- 既存のクラス／関連線／書き出しの挙動を壊さない（回帰なし）。
- 既存の保存データ（packages を持たない旧 JSON）も読み込めること（後方互換）。

## 6. 制約・前提
- 技術: Next.js + @xyflow/react（v12）。リサイズは React Flow の `NodeResizer` を利用。
- パッケージは React Flow のカスタムノード（`type: "umlPackage"` など）として実装。
- z 順はクラスより背面（node の zIndex またはノード配列順で制御）。
- schemaVersion のメジャーは互換範囲で運用（packages 追加は加算的変更）。

## 7. 受け入れ条件（完成の定義）
- 「パッケージ追加」でフォルダ型の枠が現れ、ドラッグで移動できる。
- 選択するとリサイズハンドルが出て、角・辺で大きさを変えられる。
- 枠は複数クラスの上に重ねても、クラスをクリックすればクラスが選択できる（枠が背面）。
- Inspector でパッケージ名を編集でき、タブに反映される。
- 選択して削除できる。
- 図を保存→再読み込み（リロード）してもパッケージの位置・サイズ・名前が保たれる。
- SVG / PNG 書き出しにパッケージ枠が含まれる。
- 既存のクラス・関連線・各書き出しに回帰がない。

## 8. 未確定・要確認事項（承認前に確認）
以下は既定で進める想定。異論があれば指摘してほしい。
- 名前ラベルは **タブ内に表示**（本体上部ではなく）。
- 本体の塗りは **ほぼ透明**（極薄の色 or 無地）にして背面のクラスを隠さない。
- リサイズは **選択時のみ** ハンドル表示（React Flow `NodeResizer` の既定）。
- 追加時の既定サイズは概ね **320×200 程度**（あとで手動調整）。
- ネスト・自動フィット・真のグループ化は将来検討（本要件では Out）。
